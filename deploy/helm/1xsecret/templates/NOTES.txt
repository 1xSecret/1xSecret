3. Need a production Postgres? This chart deliberately does not bundle
   one — pair it with the CloudNativePG operator instead:

  apiVersion: postgresql.cnpg.io/v1
  kind: Cluster
  metadata: { name: db, namespace: databases }
  spec: { instances: 3, storage: { size: 10Gi } }

   CloudNativePG generates a Secret `db-app` whose key `uri` holds the
   full connection string. Point this chart at it:

  database:
    existingSecret:
      name: db-app
      key: uri

   Note: if the database lives in another namespace than the app you must
   copy/replicate the Secret into the app namespace, and the in-cluster
   DNS name inside the URI must be namespace-qualified
   (e.g. db-rw.databases.svc.cluster.local).
